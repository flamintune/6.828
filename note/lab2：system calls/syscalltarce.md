# syscall tracing

## 实现trace追踪系统调用
- trace 接受一个整数（掩码），来指定需要来追踪哪些系统调用，例如read的调用是5，则 整数如果 10000 第五位为1的话就要追踪该系统调用，实现方法就是在proc中声明一个变量来保存该掩码，然后在每一次系统调用完成后打印输出就好了

## 系统调用 在 xv6 里面的添加流程
- 首先在 user/user.h 中 添加 函数定义
- 然后在 user/user.pl 中添加 一个 stub
  - 这里有个迷惑之处，就是什么是 stub ，查了一下发现，stub指的是 现有系统依赖的一个替代，可以在不去处理依赖的情况下直接测试自己的代码
- 在kernel/syscall.h 添加系统调用号
- 最后不要忘了在 kernel/syscall.c中的 一个用于存放系统调用号到系统调用的映射中添加trace的映射
